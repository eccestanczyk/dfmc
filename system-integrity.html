The local save is a disposable prediction cache, overwritten on every server reconciliation — editing it is inert, not forbidden. The transaction contract

Every economic action is: check server balance → atomic debit + grant → or reject whole. Nothing partial, nothing client-computed. Two mechanisms make "atomic" real rather than aspirational:

MechanismWhat it stops
Idempotency keysEvery request carries a unique key; replays of the same request are no-ops. Retried packets and deliberate double-sends cannot double-grant.
Version tokens (write locks)Cloud Save writes are compare-and-swap on a version token. Two simultaneous spends that both read balance 100 cannot both commit — the second fails and re-reads. This is the actual historical dupe vector: Diablo 3's trillions-of-gold incident was a transaction-layer race in its own auction house, not a hacked client.

Reward path & the accrual cap

The client reports what happened (battle event logs); the server derives reward values from its own tables — a client cannot name a number. On top of that sits the single cheapest, strongest control an autoplay game can have:

Wall-clock accrual cap. Rewards credited per flush are capped by elapsed real time (~30 battles/hour at autoplay pace, with headroom). A client claiming 4,000 battles in ten minutes is not investigated — it is arithmetically impossible and clipped at the ledger. Farming faster than time permits cannot be expressed in the protocol.

Verification — trust, but replay

EventVerification
Boss kills, floor/ladder advances100% seeded-replay verified: the server issued the encounter seed, the client submits the move log, Cloud Code re-simulates deterministically and compares the claimed result. Value-gating events are never taken on faith.
Ordinary grind battlesRandom sampling + statistical invariants: monotonic XP, battle-rate plausibility, streak-threshold consistency, drop-rate bounds, currency reconciliation. Cheap to run, expensive to fool at scale.
Rated PvP matchesEvery match ends with one deterministic replay adjudication — full spec on PvP, Timers & Persistence (team legality by recompute-from-inputs, commit–reveal rounds with reveal-derived RNG seeds, per-round state checksums, replay as final authority).

The residual threat: farm accounts

Once client cheating is inert, the economy's remaining attacker plays legitimately — fleets of accounts running real autoplay to print tradeable items. Mitigations, in order:

GateEffect
Steam's $5 limited-account gateMarket and trading are locked behind Steam's own spend threshold — each farm account has a real cost floor.
Progression-gated market unlockTrading unlocks only past a progression milestone; a farm account must invest real climb-time before it can sell anything.
Listing velocity monitoringAccounts whose listing rate is inconsistent with their play pattern get flagged for the audit queue — the CS:GO key-fraud lesson: watch the exit, not just the faucet.

Failure & cost doctrine

Precedent says the model holds: the mobile-strategy pattern this follows (client as pure view, all authority server-side) is why modded clients achieve nothing against such games. Its known failure modes are the transaction layer (handled above) and cost blowout — the naive build of this same architecture (UGS Economy ledgers, per-round PvP server resolution) priced at >$450k/month at 500k CCU; the committed build lands at ~$0.02–0.03 per concurrent user per month (~$12–15k/month at 500k CCU), dominated by batched Cloud Save writes. Flush tiers: immediate for stakes (deaths, trades, ladder), batched for grind accrual.
